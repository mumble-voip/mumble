# Настройка TFAR в mumble-tfar (Arma 3: Storm of The Galaxy)

Этот клиент — Mumble со встроенной поддержкой TFAR, полная замена связки
TeamSpeak 3 + TFAR-плагин. Ниже — инструкции для игроков, администраторов
сервера и мишн-мейкеров.

---

## Для игроков

### Установка

1. Скачайте свежий релиз: <https://github.com/StormofTheGalaxy/mumble-tfar/releases/latest>
   - `mumble_client-*.exe` — установщик (рекомендуется), **или**
   - `mumble-tfar-*-win64-portable.zip` — распаковать в любую папку.
2. Установите мод **@task_force_radio** в Arma 3 (Steam Workshop / репозиторий проекта) —
   тот же мод, что и для TeamSpeak. **TS3-плагин ставить не нужно**, TeamSpeak можно удалить.
3. Запустите `mumble.exe`. При первом запуске пройдите стандартный мастер настройки
   аудио Mumble (микрофон, режим передачи).

### Подключение

Сервер проекта **stormofgalaxy.com** уже добавлен в избранное
(«Сервер → Подключиться → Избранное»). Либо: «Настройки → TFAR → Подключиться к серверу».
Ник укажите любой — при входе в игру клиент автоматически покажет всем ваше игровое имя.

### Проверка работы

1. Подключитесь к серверу Mumble, зайдите в игру на миссию с TFAR.
2. В чате Mumble появится **«TFAR: Arma 3 connected»** и прозвучит сигнал.
3. «Настройки → TFAR» — статус: `Connected to Arma 3: Yes`, `Playing: Yes`.
4. Клиент сам переместит вас в serious-канал миссии (если он настроен).

### Настройки по умолчанию (применяются при первом запуске)

- Режим передачи — **голосовая активация (VAD)**;
- Push-to-Talk дополнительно привязан к клавише **`` ` `` (ё)** — работает поверх VAD:
  зажали — микрофон открыт принудительно;
- Стандартные звуковые сигналы включения/выключения микрофона **отключены**
  (вернуть можно в «Настройки → Аудиовход → Audio cue»).

Все значения — только стартовые: меняйте что угодно, ваш выбор сохранится.

### Рекомендованные настройки звука

- «Настройки → Аудиовыход»: **частота дискретизации устройства 48000 Гц** (как в TS3;
  при других значениях эффекты радио будут слегка неточны).
- Позиционный звук Mumble (Positional Audio) для Arma **не нужен** — TFAR делает
  собственную 3D-обработку. Плагины позиционного звука для Arma не включайте.
- Режим передачи — любой: и Push-to-Talk, и голосовая активация полностью
  поддерживаются; тангенты радио из игры работают поверх них.

### Если что-то не работает

| Симптом | Что проверить |
| --- | --- |
| Нет «Arma 3 connected» | Мод @task_force_radio включён? Игра и Mumble запущены от одного пользователя? |
| Слышно всех «плоско», без 3D | Вы в игре? (`Playing: Yes` в настройках TFAR). Все ли в одном канале Mumble? |
| Не слышно игрока | У него этот же клиент? Обычный Mumble без TFAR в serious-режиме мьютится. |
| Радио без эффектов/звуков | Папка `tfar/radio-sounds` рядом с `mumble.exe` (или путь в «Настройки → TFAR»). |
| Диагностика | В чат: `/tfar diag` (общее), `/tfar pos` (позиции), `/tfar rstflt` (сброс фильтров), `/tfar full` (дамп логов в `~/mumble-tfar-logs/`). |

## Для администраторов сервера

### Рекомендуется: сервер mumble-tfar (с TFAR-расширениями)

В каждом релизе есть `mumble-tfar-server-*-linux-x86_64.tar.gz` — наш murmur
с серверными TFAR-расширениями (`tfarsupport=true` в `mumble-server.ini`,
включено по умолчанию). Что дают расширения:

- **радио-трафик не попадает под рейт-лимиты** — на ванильном сервере лимит
  4 plugin-сообщения/сек на игрока, при активном радиообмене события нажатия/
  отпускания тангенты могут теряться (зависшие передачи);
- **канальная рассылка** — сервер сам раздаёт радио-команды всем в канале
  (меньше трафика, нет гонок с локальным списком участников);
- **серверный кэш состояния** — сервер помнит игровой ник/статус каждого клиента
  и сам выдаёт их входящим в канал; клиенты не рассылают состояние каждые 4 секунды.

Клиент показывает режим в «Настройки → TFAR → Server TFAR extensions»
и в `/tfar diag` (`serverTFARSupport`).

### Альтернатива: любой ванильный Mumble server ≥ 1.4.0

Работает в режиме совместимости без модификаций. В этом случае поднимите лимиты
в `mumble-server.ini`: `pluginmessagelimit=30`, `pluginmessageburst=60` —
TFAR шлёт сообщения при каждом нажатии тангенты и обновлении состояния.

### Общее

1. Создайте канал для игры (serious-канал), например `Storm — In Game`.
   Пароль канала реализуется **access-токеном**: клиент передаст значение
   `TF_serious_channel_password` как токен при входе.
2. Каналов может быть несколько (лобби, штаб и т.д.) — TFAR работает в том канале,
   где находится игрок; авто-перемещение настраивается миссией.

## Для мишн-мейкеров

Все настройки TFAR задаются как обычно — через мод/миссию (CBA settings),
клиент получает их командой `SETCFG` автоматически. Важные моменты:

- `TF_serious_channel_name` — **имя канала Mumble** (вместо имени канала TS).
  Вариант `ID:<число>` — это ID канала, который клиент сообщает игре
  (`TS_INFO CHANNELID`, также виден в `/tfar diag`; равен ID канала Mumble + 1).
  Рабочий процесс тот же, что с TS: зашли в нужный канал → спросили CHANNELID →
  прописали в конфиг.
- `TF_serious_channel_password` — передаётся серверу Mumble как access-токен.
- Остальные параметры (`intercomVolume`, `spectatorCanHearFriendlies`,
  `moveWhileTabbedOut`, `muteSpectators` и т.д.) работают без изменений —
  слой совместимости поддерживает весь набор настроек TS-плагина.

## Обновления

Клиент при запуске проверяет свежие релизы в
[StormofTheGalaxy/mumble-tfar](https://github.com/StormofTheGalaxy/mumble-tfar/releases).
Если найдена новая версия — **предлагает скачать и установить её автоматически**
(клиент скачает инсталлятор, запустит его и закроется). Отключить автообновление
можно галочкой в «Настройки → TFAR» — тогда о новой версии будет только сообщение
в чате со ссылкой. Ручная проверка — «Настройки → TFAR → Проверить обновления»
или меню «Справка → Проверить обновления».
Проверка обновлений ванильного Mumble отключена (она предлагала бы сборку без TFAR).
